Project

Requirements
•	Public submission form with: Name, optional Email, Category (Bug, Feature Request, Improvement, Question, General Feedback), Priority (Low/Medium/High), Title, Description, optional Screenshot upload, and a confirmation message.
•	Password-protected Admin Dashboard.
•	View, search, and filter submissions.
•	Update ticket status (New, Reviewing, In Progress, Completed, Closed).
•	Add internal notes and delete submissions.
•	Dashboard summary cards for Open Tickets, Completed Tickets, High Priority, and New This Week.
•	Responsive design that works well on desktop and mobile.
•	Build in Replit using the technologies you feel are most appropriate.
Deliverables

•	Provide a short README with setup instructions.
•	Record a 5–10 minute walkthrough explaining your design decisions, improvements you made, and what you would build next.
